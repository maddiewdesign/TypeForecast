1. Weather Data (Fetching live information)

The first part connects to a weather API and pulls in real-time data such as:

  • wind speed and direction
  • temperature
  • chance of precipitation

This data is updated every time the function runs. If the system is in "sandbox mode," it ignores live data and uses preset values instead. Once the data is received, the interface is updated to reflect the new weather conditions.

//Data fetching:

function fetchWeather() {

  let url = `https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lon}
&hourly=precipitation_probability¤t_weather=true&windspeed_unit=mph`;

  fetch(url)
    .then(r => r.json())
    .then(data => {

      if (!window.sandboxActive && data.current_weather) {
        windSpeed = data.current_weather.windspeed || 0;
        windDirection = data.current_weather.winddirection || 0;
        temperature = data.current_weather.temperature || 0;
      }

      if (!window.sandboxActive && data.hourly) {
        precipitation = data.hourly.precipitation_probability[0] || 0;
      }

      updateUI();
    });
}

2. Rain System

The rain effect is created by scanning across the screen in a grid.

At each point:

  • a noise pattern decides whether "rain" should appear there
  • a density setting controls how heavy or sparse the rain is
  • only areas that overlap with the text image are drawn

When rain appears, it is shown as small circles. The size of each circle depends on how strong the rainfall is in that location, and the colour comes from the temperature.

In short: rain is simulated as animated dots that appear more or less densely depending on weather data and noise patterns.

//Rain

noStroke();

for (let y = 0; y < height; y += cell) {
  for (let x = 0; x < width; x += cell) {

    let rain = noise(x * 0.01, y * 0.01, t);
    let threshold = map(density, 0, 1, 0.7, 0.25);

    let intensity = constrain(map(rain, threshold, 1, 0, 1), 0, 1);
    let d = intensity * cell * 0.9;

    if (d > 0 && txtImg.get(x, y)[0] > 128) {
      fill(getTempColor(x, y));
      ellipse(x + cell/2, y + cell/2, d, d);
    }
  }
}

3. Wind System

The wind is visualised using angled line segments spread across the screen.

For each line:

  • noise creates variation, making wind more realistic
  • thickness changes depending on gust strength
  • only certain areas are drawn based on a density threshold
  • lines only appear where the text image allows it

Each line follows a direction influenced by wind data, creating the impression of flowing air moving through the composition.

Summary: wind becomes moving strokes that vary in strength, direction, and density.

//Wind:

for (let i = -height; i < width + height; i += 18) {

  let gust = noise(i * 0.01 + t);
  strokeWeight(0.6 + pow(gust, 2) * maxThickness);

  for (let j = -width; j < width * 2; j += 14) {

    if (noise(j * 0.02 + t * 1.8, i * 0.01) > densityField) {

      let x1 = width/2 + px*i + dx*j;
      let y1 = height/2 + py*i + dy*j;

      if (txtImg.get(x1, y1)[0] > 128) {
        stroke(getTempColor(x1, y1));
        line(x1, y1, x1 + dx*14, y1 + dy*14);
      }
    }
  }
}

4. Colour System

Colour is controlled by temperature.

Here's how it works:

  • the current temperature is matched to a colour range (cold → warm spectrum)
  • the system blends between two nearest colours for smooth transitions
  • final colour is adjusted slightly using noise so it shifts gently across the image

Summary: temperature directly determines the colour mood of the entire visual system.

//Colour:

function getTempColor(x, y) {

  let temp = window.sandboxActive
    ? sandboxValues.temperature
    : temperature;
  temp = constrain(temp, -15, 45);

  let lower = colors[0];
  let upper = colors[colors.length - 1];

  for (let i = 0; i < colors.length - 1; i++) {
    if (temp >= colors[i].t && temp <= colors[i + 1].t) {
      lower = colors[i];
      upper = colors[i + 1];
      break;
    }
  }

  let amt = (temp - lower.t) / (upper.t - lower.t);
  let base = lerpColor(lower.c, upper.c, amt * (0.7 + 0.3 * n));

  colorMode(HSB, 360, 100, 100);

  let h = (hue(base) + map(noise(x*0.01,y*0.01,t),0,1,-20,20)) % 360;
  let s = saturation(base);
  let b = brightness(base);
  let final = color(h, s, b);

  colorMode(RGB, 255);
  return final;
}

Summary

Instead of drawing static graphics, the code turns weather data into a living system:

  • rain = probability + noise
  • wind = direction + flow + variation
  • colour = temperature gradient

The result is a visual language where weather becomes the generator of form
